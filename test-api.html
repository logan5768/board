HTTP <статус> — сервер ответил. Дальше смотрите тело ответа (JSON с ошибкой или с content[0].text).

TypeError: Failed to fetch — почти всегда CORS-блок браузера: сервер не вернул нужные заголовки Access-Control-Allow-Origin / -Headers / -Methods в ответ на preflight (OPTIONS). У основного приложения тот же симптом «Сеть недоступна».

Как лечится: 1) включить CORS-прокси из списка выше; 2) обратиться к провайдеру (zenoid), пускают ли они браузерные запросы и какие заголовки разрешают; 3) запускать страницу через локальный прокси, который сам отправляет запрос на API и возвращает ответ.

Эквивалентный curl сгенерируется после отправки
